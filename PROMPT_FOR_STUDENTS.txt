## WHAT IS THIS SYSTEM

This is a "Tourism Promotion Platform" — a full-stack web application for promoting tourism in the Ogooué-Maritime Province of Gabon, Africa.

It is an INFORMATIONAL platform (like a government tourism website). It is NOT an e-commerce site, NOT a booking system, and does NOT require user registration or payments. Think of it like VisitRwanda.com or a digital tourism brochure.

The system has TWO users:
- VISITORS (tourists) — they browse the website to discover places
- ADMIN (platform manager) — they login to manage all the content

---

## HOW THE SYSTEM IS BUILT (Technical Architecture)

### Frontend (What users see):
- Built with: HTML, CSS, JavaScript (no framework)
- Pages: Homepage, Tourist Sites, Hotels, Events, Tour Guides, Favorites, Contact, Admin Login, Admin Dashboard
- How it works: Each page has a JavaScript file that calls the backend API using fetch(), receives JSON data, and dynamically builds HTML to display it

### Backend (The server):
- Built with: Node.js + Express.js
- Pattern: MVC (Model-View-Controller)
- It provides a REST API with 15+ endpoints
- Handles: routing, authentication, file uploads, PDF generation
- Middleware: JWT token verification (auth.js), file upload handling (Multer)

### Database:
- MongoDB with Mongoose ODM
- 6 Collections:
  - admins — email, passwordHash
  - sites — name, category, description, city, coordinates, images, createdAt
  - hotels — name, priceRange, phone, whatsapp, address, city, images, createdAt
  - events — title, date, venue, description, city, posterImage, createdAt
  - guides — fullName, languages[], fees, phone, areasCovered[], city, createdAt
  - inquiries — name, email, message, createdAt

### Authentication:
- Admin passwords hashed with bcrypt (10 salt rounds)
- Login returns a JWT token (expires in 24h)
- All admin routes protected by auth middleware that verifies the token

### File Uploads:
- Multer middleware handles image uploads
- Images stored on server disk in /uploads folder
- File paths saved in database

### PDF Reports:
- PDFKit library generates reports server-side
- Queries all collections by date (daily = today, weekly = last 7 days)
- Creates formatted PDF with title, statistics table, recent inquiries list
- Streams PDF directly to browser as file download
